⚙️ 這個網站是怎麼做出來的？

一趟旅行、一份行程表，透過 AI 工具協作與前端開發，
從 Word 文件變成可分享的 GitHub Pages 網站。

製作流程
1
取得原始行程資料
從雄獅旅遊官網擷取行程單，以及領隊提供的 Word 版行前說明，作為所有內容的資料來源。
雄獅旅遊行程單Word 行前說明文件
2
AI 彙整與資訊補強
將行程單上傳至 Claude，請它解析每日行程、航班資訊、飯店位置，並對照 Google Maps 確認各店家距飯店的步行時間，以及4月氣溫與穿搭建議。
Claude (Anthropic)Google Places API
3
採買策略研究
將紐西蘭伴手禮相關部落格文章上傳至 NotebookLM，彙整「哪些在超市買」、「哪些在藥妝買」、「哪些在專賣店買」，產生結構化比較表。
NotebookLM (Google)旅遊部落格文章 ×3
4
前端開發：單頁式網站
以 Claude 輔助撰寫 HTML/CSS/JavaScript，實作分頁導覽、Lightbox 圖片放大、每日備忘跳轉採買地圖的錨點連結，以及手機響應式排版。全部打包為單一 index.html，不依賴任何框架。
HTML5 / CSS3Vanilla JavaScriptSVG 地圖手繪Claude 輔助開發
5
部署至 GitHub Pages
建立 GitHub Repository，上傳 index.html 並開啟 Pages，讓爸爸與朋友不需登入帳號、直接以手機瀏覽器開啟分享連結即可閱讀全部內容。
GitHub PagesGit Version Control
技術亮點
📱
手機優先設計
Tab 橫向捲動、響應式表格、Lightbox 放大圖最大化至 94vw，在 6.6 吋手機上完整呈現。
🔗
跨分頁錨點導航
每日備忘的「採買提醒」點擊後自動切換至採買地圖 Tab，並捲動至對應店家位置，黃色高亮提示。
🖼️
Lightbox 無框架實作
純 JavaScript 實作圖片放大遮罩，修正 iOS 長按系統選單問題，點任意處關閉，不依賴套件。
🗺️
SVG 手繪路線圖
以 SVG path 手繪南北島輪廓與旅行路線，呈現「8字型」動線，不依賴地圖 API。

💬 製作心得
這份網站的起點只是一份 Word 行程表，透過 AI 工具協作，最終變成任何人都能用手機直接瀏覽的旅遊指南。整個製作過程大約 3–4 天，主要時間花在資訊核實與 UI 細節調整上。AI 負責彙整與產出草稿，人工負責判斷、驗證與最終決策——這種人機協作的模式，讓個人也能做出以前需要一個團隊才能完成的事情。
